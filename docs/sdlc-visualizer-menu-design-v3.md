## 1. 重构后的页面结构

```
├─ 项目工作台
│   ├─ 应用管理
│   ├─ 项目管理
│   ├─ 产物浏览器
│   └─ 审批中心
│
├─ 需求设计室
│   ├─ 脑暴室
│   ├─ 需求方案          ← 合并：概要需求 + 详细需求
│   ├─ 需求草图
│   └─ 需求确认
│
├─ 方案设计室
│   ├─ 设计方案          ← 改名：设计文档
│   ├─ 系统结构          ← 原 C4 架构
│   ├─ 页面布局          ← 原线框图
│   ├─ 交互原型          ← 原 OpenUI
│   ├─ 接口对照          ← 原数据绑定
│   └─ 设计定稿          ← 合并：确认提交 + 架构治理
│
└─ 开发执行室
    ├─ 任务编排          ← 改名：任务中心
    ├─ 代码开发
    └─ 测试调试
```

**总计：4 个一级菜单（室），17 个二级页面（功能点），完全扁平无三级嵌套。**

**反馈回方案设计室流程**：
```
测试调试页面发现接口偏差
    │
    ▼

    │
    ▼
弹窗：选择关联设计产物（下拉选择 HLD/DD/接口契约）
    │
    ▼
系统自动：
  1. 在方案设计室-设计定稿创建变更请求（CR）
  2. 标记该设计产物为 Stale
  3. 推送通知到项目工作台
    │
    ▼
用户进入方案设计室修改设计 → 重新基线化
    │
    ▼
回到开发执行室重新执行任务
```
