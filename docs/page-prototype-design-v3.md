### 1.3 设计原则（针对需求/设计阶段）

| 原则 | 说明 |
|------|------|
| **产出-验证-调整闭环** | 每个阶段页面必须包含：① 产物生成区 ② 产物审查区 ③ 批注/修改区 ④ 重新生成入口 |
| **阶段门控可视化** | 阶段之间的人工闸门（Gate 1 / 2.5 / 2 / 3）必须在页面内显性展示，阻塞状态实时可见 |
| **产物可追溯** | 所有生成的产物（PRD、用户故事、草图、C4 图、接口契约）必须关联版本历史，支持 diff 对比 |
| **上下文继承** | 上游阶段的产物自动继承为下游阶段的输入，不可手动复制粘贴，系统自动挂载 |
| **双态模型** | Draft 态 = 探索自由编辑；Active 态 = 受门控约束，修改需走变更流程 |

**对应代码**: `frontend/src/pages/RequirementStudio/Brainstorm/index.tsx`
**SDLC 阶段**: 需求探索
**页面目标**: 产出脑暴纪要、竞品分析、规模初估，作为后续概要需求的输入上下文。

#### 2.1.2 核心功能区

| 区域 | 功能 | 按钮/操作 | 数据流 |
|------|------|----------|--------|
| 脑暴纪要 Tab | 展示 Markdown 格式的脑暴纪要，可编辑 | [编辑] [保存] [版本历史] | `GET/POST /api/v1/requirement-studio/{projectId}/brainstorm` |
| 竞品分析 Tab | 展示竞品分析 Markdown，含对比表格 | [编辑] [保存] [重新生成] | 继承脑暴纪要上下文 |
| 规模初估 Tab | 展示规模初估卡片 + 模块工时 breakdown | [重新评估] [调整参数] | `GET /api/v1/projects/{projectId}/size-estimate` |
| 执行控制台 | 触发 brainstorming Skill | [执行] [重试] [中断] | `POST /api/v1/skill-executions` |
| 审查批注 | 对脑暴结果添加批注，驱动重新生成 | [添加批注] [携带批注重生成] | `POST /api/v1/annotations` |

#### 2.1.4 产出-验证-调整闭环

```
触发脑暴 Skill → 生成脑暴纪要
       ↓
   用户审查 → 添加批注
       ↓
   [携带批注重生成] → Skill 注入批注上下文 → 重新生成
       ↓
   确认满意 → 脑暴纪要成为"锁定"状态 → 自动继承至需求方案页面
       ↓
   规模初估通过 → 解锁 Gate 1 入口
```

```
       │ Gate 1 通过 → 解锁详细需求
       ▼
  需求方案 — 详细视图 (RequirementPlan)
       │
       │ 输入: 概要需求产物（自动继承）
       │ 产出: 详细规格、接口契约初稿、数据模型
       │ 操作: 执行 Skill → 生成产物 → 审查批注 → 携带批注重生成
       │ Gate 2.5 通过 → 解锁概要设计
       ▼
  需求草图 (SketchGallery)
       │
       │ 输入: 用户故事 page_desc
       │ 产出: 低保真 SVG 草图、页面导航树
       │ 操作: 生成草图 → 路径验证 → 审查批准
       │ 审查通过 → 同步到设计方案页面
       ▼
  需求确认 (GateCenter)
       │
       │ 输入: 当前阶段全部产物
       │ 操作: 自检 → 人工审批 → 通过/驳回
       │ 驳回 → 返回上一阶段修改
       │ 通过 → 解锁下一阶段 + 锁定基线
       ▼
  方案设计室
```

### 4.2 方案设计室数据流

```
┌─────────────────────────────────────────────────────────────────────┐
│                        方案设计室 数据流                              │
└─────────────────────────────────────────────────────────────────────┘

  设计方案 — 概要设计 (DesignPlan)
       │
       │ 输入: 详细需求产物（自动继承）
       │ 产出: HLD、C4 L1/L2、技术栈选型
       │ 操作: 执行 Skill → 编辑产物 → 保存版本
       │ Gate 2 通过 → 解锁详细设计
       ▼
  系统结构 (C4Navigator)
       │
       │ 输入: C4 DSL
       │ 产出: SVG 架构图、层级视图
       │ 操作: 编辑 DSL → 实时渲染 → 关系同步 → 孤立节点处理
       │ 保存 → 同步到设计方案产物
       ▼
  页面布局 (WireframeCanvas)
       │
       │ 输入: C4 Container 图
       │ 产出: 线框图页面、导航关系
       │ 操作: 从 C4 生成 → 手动调整 → 跳转关系验证
       │ 保存 → 同步到设计方案产物
       ▼
  设计方案 — 详细设计 (DesignPlan)
       │
       │ 输入: 概要设计产物 + C4 L2 + 线框图
       │ 产出: DD、C4 L3、API 规格、OpenUI 规格
       │ 操作: 执行 Skill → 编辑产物 → 保存版本
       ▼
  交互原型 (OpenUIPreview)
       │
       │ 输入: OpenUI 规格
       │ 产出: 高保真 HTML 页面
       │ 操作: 生成原型 → 多 viewport 预览 → 审查反馈
       │ 保存 → 同步到设计方案产物
       ▼
  接口对照 (BindingPanel)
       │
       │ 输入: C4 接口 + Wireframe 页面 + OpenUI 页面
       │ 产出: 覆盖率报告、缺失/冗余/差异列表
       │ 操作: 开始检测 → 审查差异 → 回写缺失 → 同步差异
       │ 全部匹配 → 接口契约定稿
       ▼
  设计方案 — 接口契约 (DesignPlan)
       │
       │ 输入: 接口对照结果
       │ 产出: 最终 OpenAPI 3.0 YAML
       │ 操作: 编辑 YAML → 校验一致性 → 版本锁定
       ▼
  设计定稿 (DesignFinalization)
       │
       │ 输入: 全部设计产物
       │ 产出: 基线锁定产物、架构健康报告
       │ 操作: 锁定基线 → 架构扫描 → 修复问题 → Gate 3 审批
       │ Gate 3 通过 → 解锁开发执行阶段
       ▼
  开发执行室
```
