# 概要需求页面原型设计

> **页面名称**: RequirementOutline（概要需求）
> **路径**: `/requirement-studio/requirement-outline`
> **定位**: 需求对齐阶段 — 产出用户故事、PRD 概要、草图、验收标准初稿

## 2. 左侧面板 — 阶段任务树

### 2.1 树结构

| 任务节点 | 来源 | 状态 | 操作 |
|---------|------|------|------|
| 脑暴纪要 | 继承自 Draft 阶段 | `INHERITED`（只读，灰色） | [查看] |
| 用户故事 | 本阶段 Skill 生成 | `NOT_STARTED` | [执行 Skill] |
| PRD（概要） | 本阶段 Skill 生成 | `NOT_STARTED` | [执行 Skill] |
| 草图 | 本阶段 Skill 生成 | `NOT_STARTED` | [执行 Skill] |
| 验收标准 | 本阶段 Skill 生成 | `NOT_STARTED` | [执行 Skill] |

### 2.2 节点交互

- **点击节点**：中间产物区切换至对应 Tab 并显示内容
- **悬停节点**：显示节点描述 tooltip
- **脑暴纪要**：只读，点击后中间区显示脑暴文档（Markdown），无执行按钮
- **其他节点**：显示 `[执行 Skill]` 按钮，点击后右侧面板加载对应 Skill 快照

## 3. 中间产物工作区

### 3.1 Tab 切换栏

| Tab 标签 | 图标 | 内容类型 | 默认状态 |
|---------|------|---------|---------|
| 用户故事 | 👤 | 表格/卡片 | 点击"用户故事"节点时激活 |

## 5. 交互状态流转

### 5.1 任务执行流程

```
用户点击"用户故事"节点 → 中间区切换至"用户故事"Tab
  ↓
用户点击右侧面板 [执行]
  ↓
POST /api/v1/requirement-studio/{projectId}/stage/requirement-outline/execute
  Body: { skillId: "requirement-extraction", context: { input: "brainstorm-notes" } }
  ↓
右侧面板状态更新: PREP → EXEC → POST
  ↓
SSE 推送日志 → 实时日志面板更新
  ↓
执行完成 → 中间区自动刷新用户故事列表
  ↓
任务状态更新为 COMPLETED
  ↓
用户点击 [提交审查] → 进入审查流程
  ↓
审查通过 → 顶部导航条解锁"详细需求"阶段
```

### 5.2 审查批注流程

```
用户在中间区选中 PRD 文本 → 右键"添加批注"
  ↓
弹出批注输入框 → 输入审查意见
  ↓
批注显示在右侧面板"行内批注"列表
  ↓
用户点击 [携带批注重生成]
  ↓
系统自动将批注注入 Skill 上下文 → 重新执行
  ↓
新产物生成 → 批注对应问题修复
```

## 6. 数据结构

### 6.1 任务节点数据

```typescript
interface RequirementTask {
  taskId: string
  taskName: string
  taskType: 'inherit' | 'skill'  // inherit=继承, skill=本阶段生成
  status: 'inherited' | 'not_started' | 'in_progress' | 'completed' | 'failed'
  skillId?: string              // skill 类型任务必填
  outputArtifactId?: string      // 产物 ID
  outputArtifactType?: 'user-stories' | 'prd' | 'sketch' | 'acceptance-criteria'
  description: string
  createdAt?: string
  completedAt?: string
}
```

### 6.2 用户故事数据

```typescript
interface UserStory {
  id: string
  role: string
  description: string
  priority: 'P0' | 'P1' | 'P2' | 'P3'
  status: 'generated' | 'modified' | 'confirmed'
  acceptanceCriteria: string[]
  sourceTaskId: string
  createdAt: string
  updatedAt: string
}
```

### 6.3 规模初估数据

```typescript
interface SizeEstimate {
  moduleCount: number
  interfaceCount: number
  pageCount: number
  entityCount: number
  complexity: 'low' | 'medium' | 'high'
  riskLevel: 'low' | 'medium' | 'high'
  recommendedPath: 'lite' | 'standard' | 'complex'
  estimatedWeeks: number
  estimatedPersonMonths: number
  breakdown: Array<{
    moduleName: string
    estimatedHours: number
  }>
}
```

## 7. 与现有组件的复用关系

| 设计元素 | 复用组件 | 调整点 |
|---------|---------|--------|
| 顶部阶段导航 | `StageNavBar` | 无调整 |

| 底部状态栏 | `StatusBar` | 无调整 |
| 右侧执行面板 | `ExecutionPanel` | 无调整 |
| 右侧审查面板 | `ReviewPanel` | 无调整 |
| 规模初估卡片 | **新增** `SizeEstimateCard` | 本阶段特有 |

## 8. 响应式适配
