# SDLC 全过程可视化平台 UI 规格书

> 本文档定义基于 `skills/sdlc` 的软件开发全过程可视化平台的界面原型、交互流程与技术实现方案。
>
> 版本：V2.0 | 2026-05-13
> 对标参考：GitHub Projects、Jira Roadmap、Linear、CrewAI Studio

---

## 一、产品定位

SDLC 全过程可视化平台是 skill-arsenal 生态的**项目级可视化驾驶舱**，核心目标不是"可视化地创建 Skill"，而是**以 `skills/sdlc` 下 25 个 Skill 为节点，将应用开发的全生命周期完全可视化**——让每个项目从脑暴到监控的每一步都看得见、可追溯、可干预。

| 用户角色 | 核心诉求 | 平台提供的解法 |
|----------|----------|----------------|
| **项目经理** | 掌握项目整体进度，识别阻塞点 | SDLC 流程画布 + 实时健康度看板 |
| **开发者** | 清楚当前该执行哪个 Skill，查看输入输出产物 | 阶段详情面板 + 产物浏览器 |
| **Tech Lead / 架构师** | 在关键闸门把关，评审设计/需求/发布 | Gate 审批中心 + 历史对比 |
| **业务方 / 产品经理** | 不读代码也能了解开发进展和风险 | 项目级 Dashboard + 产物可视化渲染 |

```text

    ├── 阶段耗时对比（同类型项目横向对比）

### 3.2 SDLC 流程画布（Flow Canvas）—— 核心页面

画布以 `skills/sdlc` 下 25 个 Skill 为节点，按实际执行顺序拓扑布局，是平台的灵魂页面。

#### 3.2.1 拓扑图视图（默认）

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  🔷 用户中心重构    [拓扑图] [泳道] [列表]    [导出报告] [暂停流水线]        │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   需求探索                    设计                          编码实现        │

│  │brainstorm│──→┌──────┐ │   hld    │──→┌──────────┐   │ writing  │       │
│  │  🟢      │   │compet│ │  🟢      │   │ detailed │──→│  plans   │──→... │
│  └──────────┘   │-itive│ └──────────┘   │  design  │   │  🟡      │       │
│                 │ 🟢   │                │  ⬜      │   └──────────┘       │
│                 └──────┘                └──────────┘                        │
│                    │                           │                            │
│                    ↓                           ↓                            │

│              │  🟢      │              │  dev ⬜  │                        │
│              └────┬─────┘              └──────────┘                        │
│                   │ Gate-1                                              │
│                   ↓ (已通关)                                               │
│              ┌──────────┐                                                   │
│              │ detailed │                                                   │
│              │   req    │                                                   │
│              │  🟢      │                                                   │
│              └────┬─────┘                                                   │

│                   ↓ (已通关)                                                │
│                                                                              │
│   ─────────────────────────────────────────────────────────────────────    │
│   图例: 🟢 已通关  🟡 进行中  ⬜ 未开始  🔴 已阻塞  ⏸ Gate 等待审批         │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

**节点状态说明**：

| 图标 | 状态 | 含义 |
|------|------|------|
| ⬜ | 未开始 | 前置依赖未完成，不可执行 |
| 🟡 | 进行中 | Skill 正在执行，或等待 AI 返回 |
| 🟢 | 已通关 | Skill 执行成功，产物已生成，质量门禁通过 |
| 🔴 | 已阻塞 | 执行失败、质量门禁未通过、或 Gate 被 Reject |
| ⏸ | Gate 等待 | 执行到人工闸门，暂停等待审批 |

**交互细节**：
- 点击任意节点 → 右侧滑出 **阶段详情面板**
- 双击 Gate 节点 → 弹出 **审批弹窗**（sign-off / conditional / reject + 评语）
- 拖拽画布 / 滚轮缩放（参考 Figma/Linear 的交互）
- 支持"只看我负责的"筛选（按 Skill pattern 或负责人过滤）
- 并行节点（如 `interface-first-dev` + `monitoring-setup`）自动并排渲染

### 3.4 产物浏览器（Artifact Viewer）

支持对 Skill 产出的各类文档进行结构化渲染，而非纯文本阅读。

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  📂 产物浏览器 > 用户中心重构 > design > architecture.md                    │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  [目录树]                    [渲染视图]                      [源码]         │
│  📁 design/                                                                  │
│    📄 architecture.md      ┌──────────────────────────────┐                │
│    📄 tech-stack.md        │  # 系统架构设计               │                │
│    📄 data-model.md        │                              │                │
│  📁 prd/                   │  ## 1. 上下文图              │                │
│    📄 prd.md               │  ```mermaid                  │                │
│  📁 api-contract/          │  graph TD                   │                │
│    📄 openapi.yaml         │    A[用户] --> B[API Gateway]│                │
│                            │    B --> C[用户服务]         │                │
│                            │  ```                         │                │
│                            │  [▶ 渲染 Mermaid 图表]       │                │
│                            │                              │                │
│                            │  ## 2. 服务边界               │                │
│                            │  | 服务 | 职责 | 技术栈 |     │                │
│                            │  |-------|------|---------|  │                │
│                            │  | ... | ... | ... |         │                │
│                            └──────────────────────────────┘                │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

**渲染能力**：
- Markdown：标准渲染 + 代码高亮 + Mermaid 图表实时转换
- OpenAPI / Swagger：自动渲染为交互式 API 文档（类似 Swagger UI）
- 测试报告：覆盖率进度条 + 失败用例红色高亮 + 趋势折线图
- YAML/JSON：带折叠和语法高亮的结构化预览

## 四、前端技术实现方案

### 4.1 技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| **框架** | React 19 + TypeScript | 生态成熟，适合构建复杂交互式画布 |
| **状态管理** | Zustand | 轻量，适合项目级复杂状态（节点状态、审批流、产物树） |
| **UI 组件库** | Ant Design 5.x + 自定义主题 | 企业级组件丰富，支持暗黑模式 |
| **流程画布** | React-Flow (@xyflow/react) | 专为节点拓扑图设计，支持拖拽、连线、缩放、自定义节点 |
| **图表** | Apache ECharts | 阶段耗时对比、覆盖率趋势、返工热力图 |
| **Markdown 渲染** | react-markdown + rehype-highlight | Skill 产物的高亮渲染 |
| **Mermaid 渲染** | mermaid.js | 设计文档中的架构图、时序图实时转换 |
| **Swagger 预览** | swagger-ui-react | API 契约的可视化 |

## 五、与现有生态的集成

| 集成点                    | 方式                                                                                   |
| ---------------------- | ------------------------------------------------------------------------------------ |
| **skills/sdlc**        | 画布节点直接映射 `skills/sdlc` 目录下的 25 个 Skill；节点名称、图标、描述读取自各 Skill 的 `SKILL.md` Frontmatter |
| **skill-flow.yaml**    | 画布布局与连线由 `skill-flow.yaml` 的 `stages` 定义驱动；DAG 解析后渲染为拓扑图                             |
| **progress-tracker**   | 自动同步各阶段进度到看板；`progress-tracker.yaml` 状态变更实时反映在节点颜色上                                  |
| **human-decisions.md** | Gate 审批记录双向同步：平台内审批自动追加到 `human-decisions.md`；手动编辑文件后平台自动刷新状态                        |
| **validate.py**        | 阶段产物提交时自动调用 `validate.py` 进行格式合规性检查，结果展示在"质量门禁"区域                                    |
| **convert.py**         | 产物浏览器中支持一键导出当前阶段产物为 Cursor `.mdc` / Aider `CONVENTIONS.md` 格式                        |
| **Git 仓库**             | 项目产物直接存储在 Git 仓库中（如 `openspec/changes/{change}/`），平台通过 Git API 读取文件树和提交历史            |

## 六、非功能性需求

| 维度 | 目标 |
|------|------|
| **性能** | 首屏加载 < 2s（Lighthouse 性能分 ≥ 90）；25 个节点拓扑图交互帧率 ≥ 60fps；产物渲染 < 500ms |
| **实时性** | Skill 状态变更端到端延迟 < 3s（从 AI 执行完成到画布节点变色）；Gate 审批通知推送 < 1s |

| **移动端适配** | 审批中心、项目列表支持移动端查看；画布支持手势缩放（双指捏合） |
