# 正则表达式构建与解释器

## 适用场景
- 文本解析和数据提取

## 输入
- 想匹配或提取的内容
- 正则引擎
- 示例字符串

## 工作流程
1. 明确匹配目标。
2. 生成可读正则。
3. 分段解释。
4. 给出正例和反例。
5. 说明限制。

## 输出格式
- 正则表达式
- 分段解释
- 3 个匹配示例
- 2 个不匹配示例
- 限制说明

## Gotchas
- **可读性优先**：优先可读性。复杂的单行正则（如 `(?:(?=.*\d)(?=.*[a-z])...)`）应拆分为多行注释版（Python `re.VERBOSE` 风格）或给出分步解释。
- **避免过度复杂**：不必要时不使用复杂前后查找。很多场景可以用简单的字符类或分组替代，过度使用 lookaround 会降低性能和可维护性。
- **引擎兼容**：说明适用的正则引擎。JavaScript、Python、PCRE、Go 的正则语法存在差异（如后向引用、原子组），必须在输出中标注测试过的引擎版本。
