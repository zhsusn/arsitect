# 调试助手

## 适用场景
-  bug 排查和线上问题复盘

## 输入
- 错误信息
- 堆栈
- 相关代码
- 运行环境

## 工作流程
1. 解释错误表面含义。
2. 定位最可能出错位置。
3. 分析变量、类型和执行流。
4. 给出最小修复。
5. 补一条预防建议。

## 输出格式
- 现象
- 根因候选
- 定位证据
- 修复方案
- 预防建议

## Gotchas
- **证据优先**：不在上下文不足时强行下结论。若用户只提供了错误日志而无相关代码，应列出"需要补充以下信息"清单，而不是猜测根因。
- **概率排序**：多个可能原因按概率排序。给出修复方案时，先尝试影响范围最小、改动成本最低的方案，避免一上来就建议重构。
- **范围收敛**：不重写无关代码。最小修复原则：只修改与错误直接相关的代码路径，禁止借修 Bug 之名进行大范围代码风格调整。
