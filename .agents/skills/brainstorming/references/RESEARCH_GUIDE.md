# Brainstorming 资料收集指南

## 概述

资料收集是需求探索的基石。零资料禁止提问，禁止基于假设进行推断。

## 资料来源分类

### 1. 网络资料 (Web)

**触发条件**：`source_type` 为 `network` 或 `hybrid`

**执行步骤**：
1. 从变更描述中提取 1-3 个核心关键词
2. 调用 `web_search` 工具执行搜索
3. 取 Top-5 结果，生成结构化摘要
4. 标注每条资料的相关度（高/中/低）

**关键词生成策略**：
- 提取变更描述中的核心名词和技术术语
- 追加 "open source"、"best practice"、"architecture" 等扩展词
- 若涉及竞品，追加竞品名称 + "vs"

**输出格式**：
```
## 网络资料摘要

- [高相关] {标题} - {URL}
  摘要：{2-3 句话总结}
  引用价值：{技术方案/竞品功能/行业标准}

- [中相关] {标题} - {URL}
  ...
```

**无结果处理**：
- 扩大关键词（同义词扩展、英文补充）
- 仍无结果时标记 `[网络] 无相关资料`，依赖本地资料继续

### 2. 本地文档 (Local)

**触发条件**：`source_type` 为 `local` 或 `hybrid`

**执行步骤**：
1. 解析用户提供的 `@路径` 指令
2. 读取文件内容（支持 `.md`、`.txt`、`.yaml`、`.json`）
3. 提取关键段落，标注引用位置
4. 生成摘要

**输出格式**：
```
## 本地文档摘要

- {文件名}(@{相对路径})
  摘要：{关键内容总结}
  引用段落：{行号范围或章节标题}
  与本次变更的关联：{直接相关/背景参考/历史决策}
```

**文件不存在处理**：
- 提示用户确认路径
- 跳过该文件，记录到 `brainstorming-log.md`
- 继续处理其他资料

### 3. 混合模式 (Hybrid) - 默认

**执行步骤**：
1. 并行执行网络搜索 + 本地读取
2. 合并为统一资料池
3. 执行交叉验证

**交叉验证规则**：

```yaml
hybrid_validation:
  conflict_detection: true      # 检测网络资料与本地文档矛盾
  priority: "local_first"       # 冲突时以本地文档为准（项目 truth source）
  action_on_conflict: "prompt"  # 发现冲突时，向用户追加提问确认
```

**常见冲突场景**：
- 网络资料推荐技术 A，本地文档已采用技术 B
- 网络竞品具有功能 X，本地 PRD 明确排除功能 X
- 本地架构图与网络最佳实践存在显著差异
- **同一数值在不同资料中存在不同口径**（如文档 A 记录 Skill 数量 25，文档 B 记录 41）→ 必须标记为冲突，追问用户确认采用口径

**冲突处理话术（通用技术/方案版本）**：
> "我在资料收集中发现一处冲突：本地文档《{文件名}》采用 {技术A} 方案，而网络资料《{标题}》推荐 {技术B} 并已获广泛应用。请确认：
> 1. 是否维持现有 {技术A} 方案？
> 2. 是否考虑迁移至 {技术B}？
> 3. 两者是否有共存场景？"

**冲突处理话术（数值口径版本）**：
> "我在资料收集中发现数值口径冲突：本地文档《{文件名}》记录 {数值A}（{来源A}），而《{文件名B}》记录 {数值B}（{来源B}）。请确认：
> 1. 哪个口径是本次变更的基准？
> 2. 另一个口径是否作为扩展目标或替代方案保留？
> 3. 若基准口径在后续阶段被推翻（如扫描结果不符），如何处理？"

**交叉验证结果格式**：
```markdown
- [一致] {描述}
- [冲突/已确认] {描述} → 处理状态：{已确认 / 待决策}，基准口径：{数值A}，替代口径：{数值B}，采用理由：{理由}
- [冲突/待澄清] {描述} → 阻塞进入下游，需用户确认
```

## 资料融合与去重

1. **去重**：同一 URL 或同一文件段落仅保留一次
2. **分类**：将资料按主题归类（技术方案 / 竞品分析 / 用户反馈 / 项目历史）
3. **可信度标注**：
   - 高：官方文档、项目内部已确认文件
   - 中：知名技术博客、成熟开源项目
   - 低：论坛讨论、未验证的第三方文章

## 资料池输出

最终生成 `research-report.md`，包含：
- 网络资料摘要（含 URL、相关度）
- 本地文档摘要（含路径、引用段落）
- 交叉验证结果（含冲突标记）
- 可信度总览
