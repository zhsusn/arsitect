# 代码文档生成器

## 适用场景
- 技术文档补全和知识移交

## 输入
- 源代码
- 语言和框架
- 文档格式要求

## 工作流程
1. 阅读代码和调用上下文。
2. 识别功能、参数、返回值和副作用。
3. 生成文档块。
4. 补正常示例和边界示例。
5. 标出潜在异常。

## 输出格式
- 文档注释
- 使用示例
- 参数说明
- 返回值说明
- 边界情况

## Gotchas
- **代码忠实**：不编造代码不存在的行为。禁止根据函数名猜测功能，必须基于实际参数、返回值和内部逻辑生成文档，对不确定的副作用标 `[待确认]`。
- **去废话化**：不解释显而易见的废话。`getUserById(id)` 不需要写"通过 ID 获取用户"，应聚焦于边界条件、异常场景和调用前置条件。
- **示例可运行**：示例要能运行或明确是伪代码。若示例包含伪代码，必须在代码块顶部标注 `// 伪代码：仅展示逻辑，不可直接执行`。
