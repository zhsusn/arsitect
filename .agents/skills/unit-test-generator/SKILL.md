# 单元测试生成器

## 适用场景
-  Jest、Vitest、pytest、JUnit 等测试框架

## 输入
- 待测代码
- 测试框架
- 现有测试风格

## 工作流程
1. 分析公开行为。
2. 识别输入、输出和异常路径。
3. 生成正常场景测试。
4. 生成边界和错误测试。
5. 如有外部依赖，设计 mock。

## 输出格式
- 测试文件
- 测试用例说明
- 覆盖点清单
- 未覆盖风险

## Gotchas
- **行为测试**：测行为，不测内部实现细节。不要断言私有变量的状态或函数内部调用顺序，应聚焦于输入输出和公开副作用。
- **测试独立**：每个测试独立。禁止测试之间存在执行顺序依赖（如 test_B 依赖 test_A 创建的临时文件），每个测试应自带 setup/teardown。
- **顺序无关**：不依赖执行顺序。测试框架（如 Jest、pytest）可能并行运行或随机排序，测试用例必须在此前提下仍能全量通过。
