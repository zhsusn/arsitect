# CLI 会话管理 - 交互原型 {#sec-prototype}

## 1. 页面布局 {#sec-page-layout}

```
┌─────────────────────────────────────────────────────────────┐
│  [项目名]  AI CLI 终端           [Bug模式] [架构模式] [×]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  [系统] 欢迎使用 AI CLI，当前模式：Bug 修复           │    │
│  │  $ help                                             │    │
│  │  [AI] 可用命令：paste / scan / history / mode       │    │
│  │  $ _                                                │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [粘贴异常] [扫描架构] [查看历史] [清空终端] [重连]         │
└─────────────────────────────────────────────────────────────┘
```

## 2. 终端渲染区 {#sec-terminal-area}

### 2.1 输入行 {#sec-input-line}

- 左侧显示模式前缀：`bug>` 或 `arch>`。
- 支持命令历史（↑/↓ 切换）。
- 支持 Tab 键补全内置命令。
- 粘贴多行文本时保持原格式并自动发送。

### 2.2 消息气泡 {#sec-message-bubble}

| 类型 | 前缀 | 颜色 | 示例 |
|------|------|------|------|
| user | `$` | 白色 | `$ npm run build 报错` |
| ai | `[AI]` | 蓝色 | `[AI] 根因：...` |
| system | `[系统]` | 灰色 | `[系统] 正在连接...` |
| error | `[错误]` | 红色 | `[错误] 连接失败` |
| success | `[成功]` | 绿色 | `[成功] 会话已恢复` |

## 3. 模式切换控件 {#sec-mode-switch}

- 顶部 Tab 展示当前激活模式。
- 切换时弹出二次确认：当前模式有未确认操作时不可切换。
- 切换后保留系统级消息，清空模式专属上下文提示。

## 4. 快捷操作栏 {#sec-quick-actions}

| 按钮 | 模式 | 触发命令 | 说明 |
|------|------|----------|------|
| 粘贴异常 | Bug | `paste` | 读取剪贴板并格式化 |
| 扫描架构 | Arch | `scan` | 触发项目扫描 |
| 查看历史 | 公共 | `history` | 展示最近会话列表 |
| 清空终端 | 公共 | `clear` | 清空本地渲染，不影响持久化 |
| 重连 | 公共 | `reconnect` | 手动触发 WebSocket 重连 |

## 5. 状态指示 {#sec-status-indicators}

- 连接状态：在线（绿色圆点）、重连中（黄色圆点）、离线（红色圆点）。
- 当前模式：Bug 模式蓝色标签、Arch 模式紫色标签。
- 未确认操作提示：当有卡片等待用户确认时，底部显示"存在待确认操作"。
